Quick Start 
import { MCPOrchestrator } from 'mcp-orchestrator-ts';
import { OpenAIProvider } from 'mcp-orchestrator-ts/llm';

const orchestrator = new MCPOrchestrator({
  servers: {
    filesystem: {
      command: 'npx',
      args: ['-y', '@modelcontextprotocol/server-filesystem', './']
    }
  },
  llm: new OpenAIProvider({ apiKey: process.env.OPENAI_API_KEY })
});

await orchestrator.connect();

// Code Mode: LLM writes and executes code
const result = await orchestrator.generateAndExecute(
  'Find all TypeScript files and return the 5 largest by line count'
);

console.log(result.code);   // See generated code
console.log(result.result); // Get results
Why Code Mode? Traditional tool calling requires LLM round-trips for each operation. Code mode is smarter:
  • Chain operations without LLM round-trips
  • 60-75% fewer tokens for multi-step tasks
  • Inspectable code you can debug and reuse
  • Secure sandbox with timeout enforcement

LLMs are trained on millions of lines of real code, not synthetic tool-calling examples. Code mode leverages this strength for complex workflows while maintaining security through VM isolation, blocked globals, and configurable timeouts.

Code Mode: A Better Way to Use MCP

Traditional tool calling requires an LLM round-trip for each operation. Code mode lets LLMs write TypeScript code that chains multiple tool calls‚Äîinspired by Cloudflare's approach.

Traditional Tool Calling
// Each step requires LLM round-trip
User: "Find TypeScript files and count lines"

‚Üí LLM: list_directory tool call
‚Üí Execute: list_directory
‚Üí LLM: [receives ALL file data]
‚Üí LLM: read_file tool call (file 1)
‚Üí Execute: read_file
‚Üí LLM: [receives content]
‚Üí LLM: read_file tool call (file 2)
... repeat for each file ...

‚ùå Multiple round-trips
‚ùå All intermediate data sent to LLM
‚ùå Wastes tokens
Code Mode
// LLM writes code, executes once
User: "Find TypeScript files and count lines"

‚Üí LLM: Generates code
‚Üí Execute: Code runs in sandbox

const files = await tools.list_directory({path: './'});
const tsFiles = files.content.filter(f => 
  f.name.endsWith('.ts')
);

const sizes = [];
for (const file of tsFiles) {
  const content = await tools.read_file({path: file.path});
  sizes.push({name: file.name, lines: content.split('\n').length});
}

return sizes.sort((a,b) => b.lines - a.lines).slice(0, 5);

// Let LLM write and execute code
const result = await orchestrator.generateAndExecute(
  'Find all JSON files, parse them, and count total keys'
);

console.log(result.code);   // See what code was generated
console.log(result.output); // Console logs from execution
console.log(result.result); // Final result: { totalKeys: 142, files: 8 }
// Write your own code to execute
const code = `
  const files = await tools.list_directory({ path: './' });
  const jsonFiles = files.content.filter(f => f.name.endsWith('.json'));
  
  let totalKeys = 0;
  for (const file of jsonFiles) {
    const content = await tools.read_file({ path: file.path });
    const data = JSON.parse(content);
    totalKeys += Object.keys(data).length;
  }
  
  return { totalKeys, files: jsonFiles.length };
`;

const result = await orchestrator.executeCode(code, { timeout: 30000 });
// Traditional tool calling (still supported for simple tasks)
const files = await orchestrator.callTool('list_directory', { path: './' });

// Use for single-step operations
const analysis = await orchestrator.llm.generateStructured({
  schema: z.object({ summary: z.string(), count: z.number() }),
  prompt: `Analyze these files: ${JSON.stringify(files)}`
});

Snippet System: Reuse Your Best Code

Don't throw away working code. The Snippet System automatically promotes successful "Code Mode" scripts into first-class MCP tools, building a library of reusable experts.

1. Auto-Save

Instruct the LLM to add metadata, and successful executions are saved automatically.

// @name: resize_image
// @description: Resize image to specific width
// @input: {"width": "number", "path": "string"}

const sharp = require('sharp'); // Mock requirement
// ... code that does the work ...
2. Reuse Instantly

The snippet becomes a real tool in your registry, callable by name.

// Call it like any other tool
await orchestrator.callTool('resize_image', {
  path: './photo.jpg',
  width: 800
});

// Or let the LLM use it in future workflows
await orchestrator.generateAndExecute(
  'Resize all images in ./photos to 800px'
);
3. Compose Experts

Combine basic snippets into powerful workflows. The Code Mode LLM can see all your snippet-tools and orchestrate them.

// @name: process_gallery
// @description: Optimizes and uploads gallery
// Usage: automatically uses 'resize_image' and 'upload_s3' tools

const files = await tools.list_directory({ path: './raw' });
for (const file of files) {
  // Uses previously saved 'resize_image' snippet
  await tools.resize_image({ path: file.path, width: 800 });
  // Uses another snippet
  await tools.upload_s3({ bucket: 'my-gallery', key: file.name });
}

Execution Patterns

Use prebuilt patterns to keep orchestration simple. Swap steps without rewriting scaffolding.

const result = await orchestrator.execute('sequential', {
  steps: [
    { tool: 'list_directory', params: { path: './data' } },
    { llm: { prompt: 'Summarize these files', schema: summarySchema } },
    { tool: 'write_report', params: { destination: './reports' } }
  ]
});
const results = await orchestrator.execute('parallel', {
  tasks: [
    { tool: 'search_files', params: { pattern: 'error' } },
    { tool: 'run_command', params: { cmd: 'ps aux' } },
    { llm: { prompt: 'Cluster the signals', schema: clusterSchema } }
  ]
});
const result = await orchestrator.execute('conditional', {
  condition: { tool: 'check_status', threshold: 0.8 },
  ifTrue: { llm: { prompt: 'Process success result' } },
  ifFalse: { llm: { prompt: 'Handle failure case' } }
});
const result = await orchestrator.execute('retry', {
  task: { tool: 'unreliable_api' },
  maxAttempts: 3,
  backoff: 'exponential'
});
